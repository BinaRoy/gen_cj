# Design Playbook / 设计流程

## Goal
- Convert requirements into a bounded, testable implementation design.

## Required Inputs
- user intent
- constraints (platform/time/scope)
- non-goals (Out of Scope)

## Required Outputs
- architecture and boundaries
- interface evolution plan
- testability strategy
- explicit Out of Scope list

## Hard Gate
- No implementation start before Out of Scope is documented.
- No API design approval without backward-compatibility note.
